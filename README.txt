Восстановить БД проще всего через pgAdmin:

1)Зайти в pgAdmin
2)Создать БД, в которую в последующем будет скопирована БД из файла
3)ПКМ->Restore
4)Format - Custom, Filename - путь к файлу statistic
5)Ok


Можно восстановить отдельные таблицы с помощью sql-скриптов.
Прошу учесть - предполагается, что восстанавливаемые таблицы не созданы в БД.
Для восстановления таблиц "продукция сельского хозяйства", "строительство" следует использовать скрипт 	script(1).sql
Для восстановление таблиц "ВРП", "Население", "Торговля и услуги населению", 
	"Цены и тарифы", "Внешняя торговля", "Транспорт", "Финансы" следует использовать скрипт 	script(2).sql
Для восстановление таблиц "Труд", "Основные фонды" следует использовать скрипт 				script(3).sql
Для восстановление таблиц "Промышленное производство", "Уровень жизни" следует использовать скрипт 	script(4).sql
Обратить внимание! Необходимо в скриптах заменить названия таблиц и столбцов на соответствующие, 
			а также указать правильный путь до соответствующей таблицы в формате csv!


Параметры backup:

...\pg_dump.exe --file "...\\STATIS~1" 
--host "localhost" 
--port "5432" 
--username "postgres" 
--no-password 
--verbose 
--format=c 
--blobs 
--section=pre-data 
--section=data 
--section=post-data 
--column-inserts 
--load-via-partition-root 
--encoding "WIN1251" "Statistic"

Параметры Restore:
...\pg_restore.exe 
--host "localhost" 
--port "5432" 
--username "postgres" 
--no-password 
--dbname "Название созданной вами БД" 
--verbose "...\\STATIS~1"

Server: 15.2-2
pgAdmin: 4.2.2-1
